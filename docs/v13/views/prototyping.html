Prototyping

A prototype is an early version of a service that allows the team to test and explore ideas before developing them. They allow you to share ideas with users early and often, so that you can find issues with the design and make improvements based on user research.

Prototyping is important because it:

  • helps you to make sure you're building the right service for your users
  • email address and business contact details
  • helps your whole team to get a shared understanding about the future of your service
  • allows your team to explore design ideas much faster and at lower risk than using your production code

There are different types of prototypes that you should use at different stages of your design - from sketches to code prototypes using the GOV.UK Prototype Kit.

When to use prototypes

You must use prototypes in the Alpha phase of your project to explore and test different approaches to the design of the service.

Continue to use prototypes in the Beta and Live stages to explore potential new features or improvements.

Types of prototype

Prototypes can vary in style from a quick pen and paper sketch to a code prototype that works like a fully interactive website.

The main types of prototype are:

  • sketches - explore and discuss basic ideas with colleagues
  • flow diagrams - show how a user navigates through a service based on their answers (only share with colleagues)
  • wireframes - visualise an early idea and get feedback from colleagues or users
  • code - show or test a realistic version of your service that looks and behaves like GOV.UK (best for user research)

You should use the type of prototype that best meets your needs at the time.

Why we use code prototypes

Code prototypes are the best approach if you want to test realistic interactions that a user has with your service.

The GOV.UK Prototype Kit lets you build realistic code prototypes of your service that look and behave like GOV.UK.

Advantages of prototyping in code

The advantages of building prototypes in code are that you can:

  • copy code from the GOV.UK Design System to add components, styles and patterns
  • learn from the research and experience of other service teams across government

Advantages of prototypes built in code

The advantages of prototypes built in code are that:

  • you can show different pages based on a user’s answers (called 'branching') to test how users respond to your content and what happens in the flow if they get a question wrong
  • you can share them by publishing on the web
  • they look and behave almost exactly like the real thing, on mobile as well as desktop
  • it helps you design within the constraints of the web
  • it's easier for the whole team to have a shared understanding of design ideas, flows and behaviours
  • it supports common tools like Git and GitHub for collaboration and project history
  • you can use plugins, for example from department design systems, to add more components, styles and patterns

The code in a prototype does not need to meet the same standards as production code. For example, it does not need to be secure or to perform well with large volumes of traffic.

When you do move into production you need to make sure any code you use is good quality so you may not be able to use the same code. Do not just copy prototype code into production.

The point is to test ideas as quickly as possible. If an idea tests well, you may want to work with a developer to make your code a bit clearer so that it’s easier for others to work with.

Sharing code prototypes

Publishing on the web

You can share code prototypes by publishing them on the web.

The GOV.UK Prototype Kit includes instructions for publishing prototypes using a service called Heroku.

When you've uploaded your prototype, you can easily share it with others. You must protect access to your prototype with a password so that members of the public do not find it and mistake it for a real service.

You must not use the GOV.UK Prototype Kit to make live services. It's designed for prototyping and does not have the security or performance features needed for a live service.

Using code collaboration software

Multiple people can collaborate on the same prototype using software called Git. The GOV.UK Prototype Kit has guidance on how to set up Git for your prototypes.
